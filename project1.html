Schedular | Website

This group project was designed to solve the challenge of managing complex schedules and coordinating events across teams, featuring an intuitive drag-and-drop interface that enables users to create, modify, and organize appointments seamlessly. It was created using HTML, CSS and JavaScript.

Design Process

Step 1: Initial research and wireframing focused on creating an intuitive user interface that would allow users to easily visualize their schedule at different time scales (daily, weekly, monthly).
Step 2: Developed interactive prototypes to test the drag-and-drop functionality and ensure smooth event creation and modification.
Step 3: Implemented core features including event creation, time slot management, and calendar views while maintaining responsive design principles.
Step 4: Conducted user testing and refined the interface based on feedback, particularly focusing on mobile usability.

My Contributions

  • Designed the layout using Figma
  • Coded the frontend using HTML, CSS, and JavaScript
  • Architected the frontend interface using vanilla JavaScript for DOM manipulation and event handling
  • Designed and built responsive layouts that adapt seamlessly across desktop and mobile devices
  • Created an efficient data structure for storing and managing calendar events in localStorage
  • Developed custom CSS animations to enhance user interaction feedback

Outcome

The final product is a fully responsive scheduling application that allows users to efficiently manage their time commitments. Key features include making reminders, multiple calendar views (day/week/month), event categorization with color coding, and persistent data storage. The project successfully demonstrates the power of using static code like vanilla JavaScript in creating complex, interactive web applications without relying on heavy frameworks.
